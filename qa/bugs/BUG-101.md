# BUG-101: Balance displays NaN after server restart / session expiry

### Severity
- [x] S2 — High
- [ ] S3 — Medium
- [ ] S4 — Low

### Environment
| Field | Value |
|-------|-------|
| Browser | Chrome 125 |
| OS | Any |
| Viewport | Any |
| Device | Any |
| Game Version | v1.0.0 |

### Steps to Reproduce
1. Open http://localhost:3000/ (balance shows "1000")
2. Spin once (balance correctly updates)
3. Restart the backend server
4. Click SPIN again

### Expected Behavior
Either re-initialize the session gracefully, or show a clear error message. Balance should never become NaN.

### Actual Behavior
Balance displays "NaN". The spin button is still active but all subsequent spins  
also result in NaN. The game becomes unplayable.

### Root Cause
In `main.js`, `handleSpin()`:
```javascript
const result = await sweepAPI.adjustBalance(sessionId, winResult.totalWin);
balance = result.balance;  // result.balance is undefined when session expired
```
After API returns `{ error: 'Session not found' }`, `result.balance` is undefined,  
so `balance` becomes undefined. Any subsequent `balance += X` produces NaN.

### Fix
Add error handling:
```javascript
const result = await sweepAPI.adjustBalance(sessionId, winResult.totalWin);
if (result.error) {
    console.error('Session error:', result.error);
    // Re-init session
    const newSession = await sweepAPI.initSession('player1', 'fruit-frenzy', balance);
    sessionId = newSession.id;
    balance = newSession.balance;
} else {
    balance = result.balance;
}
```
