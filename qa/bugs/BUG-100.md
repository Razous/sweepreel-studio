# BUG-100: Missing asset file `lucky_7.png`

### Severity
- [ ] S1 — Critical
- [x] S3 — Medium
- [ ] S4 — Low

### Environment
| Field | Value |
|-------|-------|
| Browser | Chrome 125 |
| OS | Any |
| Viewport | Any |
| Device | Any |
| Game Version | v1.0.0 |

### Steps to Reproduce
1. Open http://localhost:3000/
2. Check console for "Failed to load asset" errors
3. Observe "seven" symbol rendering

### Expected Behavior
"seven" symbol should display its PNG asset (`lucky_7.png`)

### Actual Behavior
Engine.js symbolMap maps `seven` → `lucky_7.png`, but the actual file in `frontend/assets/symbols/` is named `sevens.png`. The asset fails to load, and the engine falls back to drawing a colored orange circle with text "SEVEN".

### Fix
Either rename `sevens.png` to `lucky_7.png`, or update engine.js symbolMap from `seven: 'lucky_7'` to `seven: 'sevens'`
